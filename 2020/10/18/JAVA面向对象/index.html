    类中的设计中两个重要的结构，属性和方法
    一、类的结构之一属性
    对比 ： 属性 vs 局部变量
    1、相同点： 定义的格式： 变量类型（int，string等） 变量名 = 变量值
    2、不同点：
    ① 在类中的位置不同
    属性；直接定义在类的一对{}内
局部变量： 声明在类的方法内，方法形参，构造器形参，构造器内部的变量
    ②关于权限修饰符的不同
    属性：可以在声明的前面，使用权限修饰符声明权限
        常见的权限修饰符有： public， private ，缺省， protected 
        没有声明的时候默认是缺省的。
局部变量：不可以使用权限修饰符
    ③ 初始化情况
    属性： 类的属性，更具其类型，都默认初始化值，我们不可以赋值
        整型（byte、short、int、long：0）
  • 浮点型（float、double：0.0）
  • 字符型（char：0 （或’\u0000’））
  • 布尔型（boolean：false）
  • 引用数据类型（例如 类、数组、接口等：null）
局部变量：没有默认初始值，所以我们在调用局部变量之前，一定给要给它赋值
                  ⭐特别的：在方法的形参列表中，是不能赋值的。这没有python的默认值
④在内存中的情况
    属性：声明在堆中（非static的属性，）
    局部变量： 声明在栈（例如main，方法下的 int i =0；这个i在栈下。又比如 student 类的，speak()的方法下的 int i =0；这个i也是在栈下的。其实main()方法也是在其对应的类下的）

二、类的结构之二方法

1、方法的声明：权限修饰符 返回值类型 方法名（形参列表）{
方法体
}
注意：在权限修饰符后会用，static， final， abstract来修饰方法（一般没得），后面细说
2、 说明
2.1权限修饰符：public ，private ，缺省， protected
一般的方法，我们默认使用public，后面讲封装性的时候，进行主要说明
2.2有返回值 vs 没有返回值
如果方法有返回值，则一定要在方法声明的时候加上返回值的类型，并在方法结束使用 return 返回的数据；（也就是说return之后，不可以有执行语句）
如果方法没有返回值，则方法声明的时候使用void来表示。
3、形参列表，方法可以声明0个，1个或多个形参
格式： 数据类型1，数据类型2，数据类型3，…
4、方法的使用：
① 可以调用当前类中的属性
② 可以调用当前类中的方法（可以调类中其它方法，也可以调自己（这叫递归）但是也要能出来，不然就死循环了）
③ 方法中，不可以定义其它方法

形参：方法定义时，声明在小括号内，就是要接受传入数据的参数；
实参：方法调用是，实际要传给形参的参数；
形参和实参可以同名

方法的重载
1、在一个类中语允许出现多个同名的方法，只要他们的形参（方法名后小括号的需要传入的叫形参）个数或者形参的类型不同即可，

俗称“两同一不同”：两同：在同一个类中，同一个方法名
                                  一不同：参数列表不同即参数类型不同，参数个数不同

⭐和方法的修饰符，返回类型，形参名，方法体都没有关系。
可变个数的形参
①具体实现的格式： 在小括号中 数据类型…变量名
②在调用方法的时候，可以输入0，1，2，3….无穷 个这个数据类型的参数
③具有可变参数的方法亦可和其它方法构成重载，只要遵守重载的要求；但是具有可变参数的方法不和具有参数类型相同的数据之间构成重载；如下是不行的，因为两个是一样的：


④可变参数的形参，只能放在形参列表的最后，且一个形参列表只能有一个可变形参。

值传递问题
在java中参数的传递机制：值传递
规则：
①如果参数是基本数据类型（那8个），此时实参赋给形参的就是实参的真实存储的数据值
⭐⭐⭐也就是说此时在栈中有实参存储一个数据值，又创建一个形参，此时把实参的数据值赋给形参，也就是说如果在方法内，形参的数据值的改变实参是不改变了，即使形参和实参的名字一样。
②如果参数是引用数据类型（string，自己创建的class等）,此时实参赋给形参的是存储数据的地址值
⭐⭐⭐也就是说，实参和形参此时虽然都在栈都有一个位子，且他们存储的都是这个引用数据类型的在堆中的地址值，也就是说他们其实是指向一个地方的，也就是说此时形参在方法中改变了，实参的值也跟着改变的。
推广：
如果变量是基本数据类型，此时赋值的是变量所保存的数据值。
如果变量是引用数据类型，此时赋值的是变量所保存的数据的地址值。

this关键字

1.可以调用的结构：属性、方法；构造器
2.this调用属性、方法：
this理解为：当前对象 或 当前正在创建的对象

2.1 在类的方法中，我们可以使用”this.属性”或”this.方法”的方式，调用当前对象属性或方法。但是，

  • 通常情况下，我们都择省略”this.”。特殊情况下，如果方法的形参和类的属性同名时，我们必须显式的使用”this.变量”的方式，表明此变量是属性，而非形参。

  • 2.2 在类的构造器中，我们可以使用”this.属性”或”this.方法”的方式，调用当前正在创建的对象属性或方法。但是，通常情况下，我们都择省略”this.”。特殊情况下，如果构造器的形参和类的属性同名时，我们必须显式的使用”this.变量”的方式，表明此变量是属性，而非形参。

3.this调用构造器：
① 我们在类的构造器中，可以显式的使用”this(形参列表)”方式，调用本类中指定的其他构造器
② 构造器中不能通过”this(形参列表)”方式调用自己
③ 如果一个类中有n个构造器，则最多有 n - 1构造器中使用了”this(形参列表)”，另外一个是super()调用的父类构造器（没写也默认存在）。
④ 规定：”this(形参列表)”必须声明在当前构造器的首行
⑤ 构造器内部，最多只能声明一个”this(形参列表)”，用来调用其他的构造器
